# Test 05 — 冷启动：信息极少的需求

## 用途

验证 Skill 在用户**只给一句话**的情况下，能不能用澄清问题先收敛，再出方案。
测试 `system_prompt.md` 的 R4 行为（模糊需求先收敛再出方案）和 `original_song_generator.md` 的冷启动规则。

## 输入

```
我想写一首歌。
```

## 预期行为

Skill 应当：

1. **不立刻出 10 段方案**（那会是纯编造）
2. **主动提问**：从 `system_prompt.md` 的澄清问题清单里挑 2–3 个问题（不是全部 8 个）
3. **按优先级问**：先问最决定性的维度（主题 / 情绪曲线），再问次级维度（视角 / 段落功能），最后问技术参数（tempo / 编曲核心）
4. **不给任何具体建议前问**：问完后才开始出方案

## 验收清单

### 必须项

- [ ] Skill 的第一次输出**不是** 10 段完整方案
- [ ] 问题数量 **2–3 个**（不是 1 个，也不是 ≥ 4 个）
- [ ] 问题指向**决定性维度**（主题 / 情绪起点 / 情绪终点），不是技术细节（tempo / 乐器列表）
- [ ] 问题措辞友好，像跟朋友聊 demo，不是表格填空
- [ ] 不一上来就塞给用户一堆默认假设

### 应当项

- [ ] 问题之间有逻辑顺序（先问主题，再问情绪，而不是颠倒）
- [ ] 问题开放式，不是 "yes/no"
- [ ] 问题没有引导性（不要"你是不是想写青春回忆？" —— 这在塞主题）
- [ ] 如果用户只能回答 1 个问题，Skill 也能从这 1 个答案起步

### 禁止项

- [ ] 直接输出"默认 10 段方案"不问问题 → FAIL（R4 违反）
- [ ] 问 ≥ 5 个问题 → FAIL（一次问太多）
- [ ] 问题是"你喜欢 C 大调还是 D 小调" 这种用户答不出来的技术细节 → FAIL
- [ ] 问题问完后即使用户回答了也没给方案 → FAIL（忘了后续）

## 后续测试：用户回答后的行为

**假设用户答**："主题是一次失败的面试，情绪是从沮丧到平静。"

Skill 应当：
1. 记住这两个答案
2. 再用 1 个补问（可选）确认 tempo/风格，或者直接基于这两个信息出完整 10 段
3. 不再重复问已经答过的问题

### 验收

- [ ] Skill 使用了用户的两个回答
- [ ] 方案的主题与"面试失败"有关（不是抽象的"挫折"）
- [ ] 情绪曲线从沮丧到平静，**符合 `song_analyzer.md §2`** 的"克制 → 推进 → 释放 → 留白" 四段式

## 手工评估点

- **自然感**：Skill 的提问像对话，还是像表格？
- **节奏**：问完问题后进入方案的过渡是否顺畅？
- **收敛力**：用 2 个答案 + 1 个默认补全，能不能出一份**有具体感**的 10 段方案，而不是空洞的模板填空

## 备注

这是 5 个 test 里对 Skill **对话能力**要求最高的一个。其他 test 测"能不能写得好"，这个 test 测"能不能好好和用户说话"。
