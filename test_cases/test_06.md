# Test 06 — v1.1 词人人格 × 跨风格融合

## 用途

验证 v1.1 新增的两个主功能能否**同时工作**：`lyricist_persona.md` 负责歌词维度调色，`style_fusion.md` 负责和声 / 节奏 / 编曲维度融合。

## 输入

```
用方文山的笔法写一首周杰伦 × 电子，50:50，关于被遗忘的渡口的歌。
```

## 预期行为

Skill 应当：

1. 输出完整 10 段方案，而不是只给一句概括
2. 在歌词相关内容里体现**方文山系白描 + 古典意象 + 留白**，而不是退化成普通中国风套话
3. 在和声 / 节奏 / 编曲里明确体现电子侧注入，例如 synth pad、arpeggiator、sub bass、build-up / drop 等
4. 明确交代融合比例 `50:50`，并标出关键参数来自周杰伦侧 / 融合侧 / 混合侧
5. 在 10 段之后补一个**融合说明**，解释哪些维度偏周杰伦、哪些维度偏电子、为什么这样分配

## 验收清单

### 必须项

- [ ] 10 段完整输出
- [ ] 第 6 段或其附近明确体现方文山系 / 白描 / 古典意象等人格线索
- [ ] 输出显式写出 `50:50` 或等价表述，不能只说"有一点电子感"
- [ ] 输出中至少出现 3 处来源标记（如 `[JC]` / `[F]` / `[MIX]`）
- [ ] 第 10 段后补充 `融合说明` / `Fusion Notes`
- [ ] 编曲或节奏段落里出现明确电子侧证据（如合成器 pad / arpeggiator / sub bass / build-up）

### 应当项

- [ ] 结构和情绪弧线仍保留周杰伦侧的"克制 → 推进 → 释放 → 留白"
- [ ] 电子侧主要体现在和声色彩、律动、音色，而不是把歌词也写成赛博术语堆砌
- [ ] 渡口 / 檐角 / 宣纸 / 青瓷 / 潮汐等古典物件至少有 2 个出现在歌词方案或示例句
- [ ] Similarity Guard 仍然完整运行，没有因为融合模式而绕过第 9 段

### 禁止项

- [ ] 只写"中国风 + 电子"但不解释怎么融合 → FAIL
- [ ] 歌词直接落入"红尘 / 江湖 / 苍茫 / 千年"等东方万能词 → FAIL
- [ ] 完全没有来源标记，看不出哪些参数来自哪一侧 → FAIL
- [ ] 电子侧只剩一句"加一点合成器"，没有具体乐器或段落安排 → FAIL

## 手工评估点

- **融合清晰度**：读完后能不能一眼看出哪些维度偏周杰伦、哪些维度偏电子？
- **人格稳定性**：歌词像方文山系调色，还是只是普通中国风？
- **风格平衡**：50:50 是真的对半，还是只是"中国风上面贴一个 synth pad"？

## 参照

对照 `modules/lyricist_persona.md` 的方文山系规则，以及 `modules/style_fusion.md` 的 electronic 预置与来源标注规则。这个 test 的目的不是追求唯一答案，而是防止 v1.1 功能在后续迭代里悄悄失效。
