# Module F — Similarity Guard（相似度自检 + 去相似化）

> 检查产出是否"过度像某首具体作品"或"过度模板化"。
> 既要 **像周杰伦风格**，又要 **不像任何一首具体歌**。
> 每次完整输出前都要跑一遍这个模块。

## 双重目标

1. **正向**：输出是否体现了周杰伦风格的关键识别特征？（Genre blend / 意象系统 / 情绪弧线 / 弱起进入 / 副歌 hook 等）
2. **反向**：输出是否过度接近某首具体作品 / 陷入套路 / 堆砌陈词？

任一方向失衡都要修正。

## 检测清单

### 1. 歌词陈词滥调检测

扫描歌词示例句，命中以下任何一条 → **BLOCK**，必须改写。

**烂梗句式黑名单（绝对禁用）**：
- "陪你到世界尽头" / "陪你到最后"
- "我的心碎成了几片" / "心碎" 作为主要动词
- "永远永远" / "永永远远"
- "让我为你唱首歌"
- "把爱藏进心底"
- "说好不哭" 类反悖句式
- "时光匆匆" / "时光飞逝"
- "一生一世"

**滥用的东方万能词**：
- 红尘 / 江湖 / 苍茫 / 刹那 / 轮回 / 宿命 / 缘分天注定
- 如果出现任何一个且不是讽刺或反用 → 警告

**抽象情感词堆砌**：
- 一节主歌里出现 3 个以上的 "孤独 / 寂寞 / 思念 / 想念 / 失落 / 难过"
- → 命中就 BLOCK

### 2. 意象密度检测

扫描每一节（Verse / Chorus 单独看），统计具体意象物件数量：

- **通过**：每节 ≤ 3 个意象物件
- **警告**：每节 4–5 个意象物件
- **BLOCK**：每节 ≥ 6 个意象物件

**组合黑名单**（同一节出现以下组合的 ≥ 4 项 → BLOCK）：
- 雨 + 车站 + 相册 + 旧街道 + 晚风 + 路灯

### 3. 旋律轮廓重复检测

检查旋律设计建议文字：

- **套路 1**：副歌首句被描述为"纯四度上行再下行"（太常见）
- **套路 2**：每段主歌都用完全相同的节奏型起句
- **套路 3**：Pre-chorus 只是 Verse 的机械移调，没有功能变化
- **套路 4**：副歌 hook 句式被描述为"十六分音符密集"（没有信息量）
- **套路 5**：Bridge 回到副歌节奏型（浪费转折位）

命中任一 → 警告 + 给出替代建议。

### 4. 段落结构模板化检测

- **BLOCK**：如果结构严格是 `V(8) - C(8) - V(8) - C(8) - Bridge(8) - C(8)`，完全没有 Pre、没有 Intro 差异、没有 Outro 差异 → 过度模板化
- **警告**：Bridge 长度 = 8 小节、段落小节数都是 8 的整数倍、没有任何非整数段

### 5. 和声进行相似度检测

检查第 4 段"和声方向"的内容：

- **套路 A**：副歌用 `I - V - vi - IV` 而没有任何借用 / 转位 / 变化 → 警告
- **套路 B**：和声进行完全用"流行标准进行家族"而没有任何色彩点缀 → 警告
- **BLOCK**：如果文本里出现"参考《XX》的和弦" / "用和《XX》一样的进行" → 绝对拒绝

### 6. hook 相似度检测

检查第 8 段"副歌 hook 构思"：

- **套路 H1**：hook 被描述为"从 5 度音跳到 1 度音长音" → 警告
- **套路 H2**：hook 被描述为"用一个纯四度跳跃锚定" → 警告
- **BLOCK**：如果 hook 描述让人立刻联想到某首知名歌曲的标志性 hook（如直接撞上《晴天》《稻香》《青花瓷》《彩虹》的 hook 位）→ BLOCK，必须换动机

### 7. 编曲描述质量检测

检查第 7 段"编曲推进方案"：

- **BLOCK**：编曲只有"钢琴进弦乐进鼓"这种流水账 → 没有信息量
- **警告**：编曲段落没有标明"什么时候加什么层"
- **警告**：没有说明 Outro 的收束方式
- **BLOCK**：副歌动态没有比主歌更丰富，或 C3 没有比 C1/C2 更饱满

### 8. 正向识别度检测

要保证最终输出**像周杰伦风格**，检查：

- [ ] 是否使用了弱起进入？
- [ ] 是否有四段式情绪弧线（克制 → 推进 → 释放 → 留白）？
- [ ] 是否有具体意象锚定抽象情绪？
- [ ] 是否使用了句尾留白？
- [ ] 是否至少用了一个借用和弦 / 色彩和弦？
- [ ] 编曲是否有明确的"打开"时刻？
- [ ] Bridge 是否和前面形成对比？

通过 ≥ 5 项 → **识别度合格**
通过 3–4 项 → **警告：风格度不足**，在第 10 段建议加强哪一项
通过 ≤ 2 项 → **BLOCK：不是华语流行**，回到模块 A 重新取参数

## 输出格式

在 10 段输出的 **第 9 段"原创性风险检查"** 里按下列格式报告：

```markdown

**歌词陈词滥调**：PASS / WARN / BLOCK — 说明
**意象密度**：PASS / WARN / BLOCK — 说明

**结构模板化**：PASS / WARN / BLOCK — 说明
```

## 去相似化策略（第 10 段内容）

根据第 9 段的命中项，在第 10 段给出具体改写方向：

### 歌词命中的改写策略

- 陈词滥调句式命中 → 给 2–3 个原创替代句
- 意象密度过高 → 指定删掉哪些意象，保留哪 3 个
- 抽象情感词堆砌 → 把这些词换成具体物件（如"孤独" → "一个人吃的便利店关东煮"）
- 东方万能词 → 换成白描句法（如"红尘" → "三两户人家的屋檐"）

### 旋律命中的改写策略

- 副歌首句套路化 → 改为"长音锚定 + 第二小节才上行"
- 节奏型重复 → 在 Verse 第 4 小节插入一次变化
- Pre 没有功能 → 加十六分音符密度 + 音区上行
- Bridge 不反向 → 给 Bridge 降到下属调或改用假声
- hook 撞车 → 换动机：从"大跳" 改为"长音拖 + 小切分收尾"

### 和声命中的改写策略

- `I - V - vi - IV` 太标准 → 换成 `I - V/vi - vi - IV`（加二级副属）
- 缺色彩 → 在 Verse 尾加一个 `iv`（借用小调）
- 副歌抬升太常规 → 改为同名大调切换
- 和声不够惊喜 → 在 Bridge 用一次那不勒斯六和弦

### 结构命中的改写策略

- 段落太规整 → 把 Pre 改成 5 小节（加 1 拍刹车感）
- Bridge 太长 → 拆成 4+4（对比型）或 6+2（酝酿+闪现）
- 没有 Intro 差异 → Intro 用非整数小节（3 或 6 小节）

### 编曲命中的改写策略

- 流水账 → 重写为"哪一拍发生什么"（如"进 C1 前半拍 Pad 进入，首拍 Drums fill"）
- Outro 没说清 → 指定逐轨减的顺序（先 Drums、再 Strings 高声部、最后剩 Piano）
- C3 不够峰值 → 加"副旋律 / 和声堆 / 弦乐高八度"任一

### 识别度不足的改写策略

- 弱起没用 → 指定 V1 首句改弱起
- 情绪曲线扁平 → 指定 Bridge 更夸张地降
- 缺借用和弦 → 指定在 Pre 插一个 `bVII`
- 没有句尾留白 → 指定副歌每句结尾空半拍

## 特殊场景：用户明确要求复制某首歌

如果用户说"写一首跟《XX》一模一样的歌" / "用和《XX》一样的和弦"：

1. **拒绝字面复制**
2. 解释原因：版权 + 去复读 Skill 的本意
3. **提供同风格替代**：用 `song_analyzer.md` 的高层规律，生成同风格但不同具体细节的原创方案
4. 明确告知用户替代方案的差异点
