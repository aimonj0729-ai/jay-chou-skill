# Module H — Style Fusion（跨风格融合）

> 允许用户将周杰伦风格与其他音乐风格/流派进行有控制的融合，输出融合方案并标注每个参数的来源。
> 与 `original_song_generator.md` 协作：Generator 负责主干流程，Style Fusion 负责注入异质参数。
>
> **核心原则**：融合不是随机混搭，是两套规则体系的有控制交叉。每个融合决策都要标明"来自哪一侧"。

---

## 融合模式

| 模式 | 用法 | 示例 |
|---|---|---|
| `genre_fusion` | 周杰伦风格 × 另一个音乐流派 | "周杰伦 × 爵士"、"中国风 × 电子" |
| `era_fusion` | 周杰伦风格 × 特定年代音乐特征 | "周杰伦 × 80s Synth-pop"、"× 90s Grunge" |
| `cross_cultural` | 周杰伦风格 × 另一种文化的音乐传统 | "华语流行 × 拉丁节奏"、"× 凯尔特民谣" |

---

## 融合比例 `fusion_ratio`

用户可以指定融合比例（默认 70:30，周杰伦侧为主）：

| 比例 | 含义 | 行为 |
|---|---|---|
| 90:10 | 几乎全是周杰伦，只有一丝异质色彩 | 只在 1-2 个维度（如节奏或音色）引入融合元素 |
| 70:30 | 周杰伦为骨架，融合侧上色（**默认**） | 结构/旋律/歌词跟周杰伦侧，和声/编曲/节奏引入融合元素 |
| 50:50 | 对半融合 | 每个维度都做融合决策，输出标注来源比例 |
| 30:70 | 融合侧为主，周杰伦只做调味 | 主干流派跟融合侧，只保留周杰伦的情绪弧线和留白原则 |

---

## 融合参数矩阵

定义每个创作维度在融合时的行为。周杰伦侧参数来自 `song_analyzer.md`，融合侧参数由本模块定义。

### 维度融合权重表

| 创作维度 | 融合难度 | 推荐策略 | 说明 |
|---|---|---|---|
| **结构** | 高 | 优先跟周杰伦侧 | 结构是骨架，两套结构混搭容易散架 |
| **情绪弧线** | 高 | 优先跟周杰伦侧 | "克制→推进→释放→留白"是核心，不轻易让渡 |
| **和声** | 中 | 可融合 | 和声是最容易融合的维度——加几个异质和弦就有效果 |
| **旋律** | 中 | 分段融合 | 主歌可以更偏融合侧，副歌回周杰伦侧保记忆点 |
| **节奏/律动** | 低 | 优先融合 | 节奏是风格辨识度最高的维度，换律动感知最明显 |
| **编曲/音色** | 低 | 优先融合 | 音色替换成本低、效果显著 |
| **歌词** | 中 | 跟词人人格 | 歌词风格主要由 `lyricist_persona` 决定，融合侧影响主题选择 |

**规则 F1**：融合比例 ≥ 50:50 时，结构和情绪弧线才允许偏离周杰伦侧。70:30 及以上，结构必须跟周杰伦。

**规则 F2**：每次融合产出必须在第 7 段（编曲推进方案）标注每个乐器/音色来自哪一侧。

## 预置融合风格库

### 1. 爵士融合 `jazz`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 七九十一十三和弦扩展；ii-V-I 进行替代部分段落；属七挂留解决；Walking bass 线 |
| 节奏 | Swing feel（三连音基底）；Brush drums 替代 pop 鼓组；反拍重音 |
| 编曲 | Upright bass 替代电贝斯；Rhodes/Wurlitzer 为主键盘；Muted trumpet 或 Tenor sax 做间奏 |
| 旋律 | 允许更大音程跳跃（六度、七度）；蓝调音（b3, b7）嵌入 |
| 歌词 | 不直接影响，但主题偏向夜晚、酒吧、雨中漫步等爵士经典场景 |
| **兼容性** | 与周杰伦轻 R&B 路线天然兼容（共享电钢、ghost note、swing 元素） |

### 2. 电子/合成器流行 `electronic`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 简化和声（Power chord / 持续低音）；侧链压缩 pump 效果 |
| 节奏 | Four-on-the-floor kick；16 分 hi-hat 序列；Build-up + Drop 结构 |
| 编曲 | 模拟合成器 pad 替代弦乐；Arpeggiator 替代钢琴分解和弦；Sub bass；Vocoder 人声 |
| 旋律 | 更短的 loop 式动机；副歌旋律更简洁重复 |
| 结构 | 可引入 Build-up(4-8) + Drop(8) 替代传统 Pre-Chorus + Chorus |
| **兼容性** | 与周杰伦电影感编曲路线部分兼容（Pad 铺底共通），但鼓组逻辑完全不同 |

### 3. 拉丁节奏 `latin`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 小调为主；Andalusian cadence (i-VII-VI-V)；弗拉门戈式属和弦 |
| 节奏 | Reggaeton dembow 律动；Bossa nova 节奏型；Clave 节奏基底 |
| 编曲 | 尼龙吉他指弹；Congas/Bongos 替代 hi-hat；Brass stabs |
| 旋律 | 更多装饰音；滑音；三连音分句 |
| **兼容性** | 与周杰伦小调都市感路线有共鸣（都偏小调），节奏差异大但可以在 Pre/Chorus 切换 |

### 4. R&B/Neo-Soul 深化 `rnb-deep`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | Maj9/min11/dim7 扩展和弦全段铺开；Gospel 色彩的 IV-iv 进行 |
| 节奏 | 更重的 groove（kick 偏移 + snare ghost 密集）；Half-time feel 段落 |
| 编曲 | Fender Rhodes 为绝对主角；Bass line 更 funky；和声人声堆叠 |
| 旋律 | Melisma（一个音节多个音）；Riff 式句尾装饰；即兴感 |
| **兼容性** | 与周杰伦轻 R&B 路线高度兼容——是"深化"而非"混搭" |

### 5. 古典/管弦乐 `orchestral`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 古典功能和声（完全终止/半终止）；对位法段落；二部/三部曲式元素 |
| 节奏 | Rubato（自由速度段落）；指挥式加速/减速 |
| 编曲 | 全编制弦乐（不只是 pad）；木管组；圆号；竖琴 |
| 结构 | 可引入古典曲式元素（呈示-展开-再现）作为 Bridge 内部结构 |
| **兼容性** | 与周杰伦电影感抒情路线天然亲和（弦乐是共通语言） |

### 6. 民谣/Indie Folk `folk`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 开放 Tuning 吉他和弦；挂留和弦（sus2/sus4）大量使用；五声音阶基底 |
| 节奏 | 指弹节奏型驱动；无鼓或极简鼓（Cajón/手鼓）；3/4 拍 |
| 编曲 | 原声吉他为绝对主角；口琴；Banjo；手风琴 |
| 旋律 | 更窄音域（六度内）；民歌式级进；副歌合唱感 |
| **兼容性** | 与周杰伦民谣清新流行路线兼容，但需要放弃 R&B 元素 |

### 7. 凯尔特/世界音乐 `world`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 调式和声（Dorian/Mixolydian）；无导音终止；Drone 低音 |
| 节奏 | 非对称节拍（7/8, 5/4）段落；民族打击乐 |
| 编曲 | Tin whistle / Uilleann pipes / Bouzouki 等民族乐器 |
| **兼容性** | 与周杰伦东方音色路线可以做"东西方民族融合"，但需要谨慎处理音色冲突 |

### 8. Trap/现代嘻哈 `trap`

| 维度 | 融合侧注入 |
|---|---|
| 和声 | 极简和声（1-2 个和弦 loop）；808 低音替代传统 bass line |
| 节奏 | Trap hi-hat 三连音；808 kick；Snare roll |
| 编曲 | 808 sub bass；Auto-tune 人声处理；Bell 音色旋律 |
| 结构 | Verse-Hook-Verse-Hook 简化结构；Drop 替代传统副歌推进 |
| **兼容性** | 与周杰伦轻 R&B 有一定共通（都是都市感），但动态曲线逻辑不同 |

## 融合执行流程

```
1. 解析融合请求
   - 识别融合风格（从预置库匹配或自由描述）
   - 确定融合比例（默认 70:30）
   - 检查兼容性评分

2. 参数分配
   - 按维度融合权重表，决定每个维度跟哪一侧
   - 标注来源：[JC] = 周杰伦侧，[F] = 融合侧，[MIX] = 混合

3. 冲突检测
   - tempo 差异 > 20 BPM → 向用户确认
   - 调性体系不兼容（如五声音阶 vs 十二平均律复杂和声）→ 建议折衷
   - 结构逻辑矛盾（如 Drop 结构 vs 传统 Pre+Chorus）→ 按融合比例决定

4. 生成融合方案
   - 在标准 10 段输出的每个维度上标注参数来源
   - 额外输出一段 "融合说明"（哪些维度做了融合，为什么这么选）

5. 融合相似度自检
   - 除了标准 similarity_guard，额外检查：
     - 融合侧是否引入了其他艺术家的标志性特征（如某个 iconic 的和声进行）
     - 两侧参数是否在实际听感上会打架
```

## 输出格式增强

融合模式下，标准 10 段输出增加以下标注：

### 在每段内标注来源

```markdown
### 4. 和声方向 Chord direction
- 主歌：`i7 - IV9 - bVII - III` [MIX: 小调都市感(JC) + 扩展和弦(jazz)]
- 副歌：`I - V - vi - IV` [JC: 流行大调抒情]
- Bridge：`ii7 - V7 - Imaj7 - vi7` [F: jazz ii-V-I 变体]
```

### 额外输出：融合说明段

在第 10 段之后（或作为补充），输出：

```markdown
### 融合说明 Fusion Notes
- **融合方案**：周杰伦轻 R&B × 爵士（70:30）
- **融合维度**：和声（混合）、节奏（混合）、编曲音色（融合侧主导）
- **保留维度**：结构（JC）、情绪弧线（JC）、歌词（JC + 词人人格）
- **关键融合点**：Bridge 使用 jazz ii-V-I 进行，间奏用 Tenor sax 替代钢琴独奏
- **风险提示**：jazz 和弦扩展可能让副歌 hook 变模糊，建议副歌保持简洁三和弦
```

## 自由描述融合（非预置库）

当用户描述的融合风格不在预置库中时：

1. 提取用户描述的关键特征（节奏型？音色？和声语言？文化背景？）
2. 在预置库中找最接近的 1-2 个作为基础
3. 补充用户描述的独特特征
4. 按标准融合流程执行
5. 标注"自由融合"并说明基于哪个预置的扩展

## 与其他模块的接口

- **输入来源**：`input_schema.json` 的 `fusion` 字段
- **影响范围**：所有创作维度（通过 `original_song_generator.md` 的参数分配层）
- **与词人人格的关系**：歌词维度主要由 `lyricist_persona.md` 决定，融合侧只影响主题选择和个别意象
- **相似度自检扩展**：`similarity_guard.md` 在融合模式下额外检查融合侧的标志性特征
