# Module G — Lyricist Persona（词人风格适配）

> 根据用户选择的词人风格，调整歌词策划方案的意象系统、句法结构、叙事手法和用词密度。
> 与 `lyric_builder.md` 协作：Lyric Builder 负责框架，Lyricist Persona 负责风格调色。
>
> **核心原则**：蒸馏词人的高层写作规律，不复制任何现有歌词。

---

## 可选词人人格

| 人格 ID | 词人原型 | 一句话风格 | 适合主题 |
|---|---|---|---|
| `fang-wenshan` | 方文山系 | 古典意象白描 + 电影蒙太奇叙事 + 精密内韵 | 东方审美、历史叙事、跨时空思念、画面感抒情 |
| `huang-junlang` | 黄俊郎系 | 超现实暗黑 + 意识流跳跃 + 哲学隐喻 | 暗黑叙事、心理惊悚、存在主义、实验性主题 |
| `jay-self` | 周杰伦自填词系 | 口语化直觉 + 生活物件 + 不修饰的真实感 | 青春恋爱、日常生活、轻松校园、直白情感 |
| `auto` | 自动匹配（默认） | 根据主题 + 情绪 + 风格参数自动选择最适配人格 | 任何 |

## 人格 1：方文山系 `fang-wenshan`

### 核心写作 DNA

**句法结构**：古典句法 × 现代语感
- 倒装句式：把动词或状语前置（"素胚勾勒出青花笔锋浓转淡" → 动作先行，物件跟随）
- 对仗与半对仗：上下句在结构上对称但不死板
- 四字格嵌入：在现代句中嵌入古典四字格（"天青色等烟雨" 的节奏感）
- 句长交错：长句叙事（10-14 字）与短句留白（4-6 字）交替

**意象选取规则**：
- 偏好具象历史物件：青瓷、宣纸、檐角、石板路、渡口、烽火台、铜镜
- 自然意象走古典路线：残荷、枯藤、暮雪、炊烟（非现代城市意象）
- 视觉优先：每句必有可画面化的物件，像电影分镜
- 通感手法：视觉写听觉（"素颜的声音"）、触觉写时间（"粗糙的掌纹"）

**叙事手法**：
- 电影蒙太奇：V1 和 V2 是两个不同时空的平行剪辑，副歌是两条线的交汇
- 第三人称旁观居多：叙述者像摄影机，不介入情绪，只拍画面
- 时间跨度大：一首歌可以跨越百年（"从前到如今"）
- 副歌点题但不说破：用意象暗示，不直接说"我想你"

**押韵策略**：
- 韵脚精密度极高：不仅尾韵，句中也有暗韵（内韵）
- 偏好鼻韵（an/en/ang/eng）：含蓄、悠远、未完感
- 错位韵使用率高：首句不押，二三句回韵
- 韵脚服务画面：选韵时优先考虑能带出意象的字

**用词密度**：
- 意象密度：高（每句 1-2 个物件），但不堆砌——每个物件都有叙事功能
- 形容词克制：用名词和动词说话，形容词只做点缀
- 文言词占比：约 15-25%（不超过，否则变文言翻译）
- 禁用现代口语：不用"超级"、"特别"、"其实"等现代虚词

### 段落分工调整

| 段落 | 方文山系偏好 | 与默认 Lyric Builder 的差异 |
|---|---|---|
| Verse | 白描长镜头，一句一景，不带情绪判断 | 默认允许情绪词，方文山系禁止 |
| Pre | 时空切换的过渡句，用一个动作连接两个画面 | 默认用情绪推进，方文山系用画面推进 |
| Chorus | 意象化的情绪升华，用物件说情感 | 默认允许直白情绪词在末句，方文山系用意象替代 |
| Bridge | 视角突变：从第三人称跳到第一人称，或时空骤变 | 默认保持视角一致，方文山系刻意打破 |

### 示例句风格参考（原创，非引用）

```
[Verse] 釉色渗进裂纹里  三月的窑火还没熄
[Pre]   隔着一道门槛的距离  你转身  我落笔
[Chorus] 把思念折成青瓷  搁在檐角等你
[Bridge] 我从画卷里走出来  才发现  雨已经停了很久
```

## 人格 2：黄俊郎系 `huang-junlang`

### 核心写作 DNA

**句法结构**：意识流 × 断裂式叙事
- 跳跃逻辑：上半句和下半句之间没有因果关系，靠情绪暗流连接
- 碎片式短句：大量 3-5 字的断句，节奏紧张
- 反常搭配：把不相关的词强行并置（制造超现实感）
- 问句密度高：反复质问，不给答案

**意象选取规则**：
- 偏好暗黑 / 哥特物件：教堂、钟塔、迷宫、面具、镜子、蜡烛、棺木、暗巷
- 身体化意象：骨骼、血管、瞳孔、指纹、心跳（不是"心"这种抽象词）
- 混合时代：中世纪 × 现代 × 超现实并存
- 颜色偏好：黑、红、灰、银（不用暖色）

**叙事手法**：
- 非线性叙事：不交代起因结果，像在读一本被打乱页码的小说
- 多重人格暗示：叙述者可能不止一个"我"
- 悬疑留白：故事不讲完，结尾比开头更模糊
- 副歌是咒语：重复、催眠、仪式感

**押韵策略**：
- 内韵 > 尾韵：句内的音节碰撞比句尾押韵更重要
- 偏好闭口韵（i/u/ü）：压抑、封闭、不安
- 刻意破韵：在该押韵的地方突然不押，制造不适感
- 叠音 / 重复：同一个词在不同句出现，像回声

**用词密度**：
- 意象密度：极高，但走超现实路线——密度本身是风格
- 动词偏暴力化：撕、裂、灼、噬、坠（不是"走/停/落"）
- 抽象词允许度高：可以用"存在"、"虚无"、"时间"——因为主题本身形而上
- 外来词混入：偶尔嵌入一个英文词或音译词

### 段落分工调整

| 段落 | 黄俊郎系偏好 | 与默认 Lyric Builder 的差异 |
|---|---|---|
| Verse | 碎片画面堆叠，像犯罪现场的证据照片 | 默认叙事线性，黄俊郎系非线性 |
| Pre | 节奏加速，短句密集，像心跳加快 | 默认用情绪推进，黄俊郎系用节奏压迫 |
| Chorus | 咒语式重复，同一句式的变体反复 | 默认求变化，黄俊郎系求催眠 |
| Bridge | 突然安静：一句长的、完整的、清醒的话 | 默认是情绪转折，黄俊郎系是清醒一瞬 |

### 示例句风格参考（原创，非引用）

```
[Verse] 镜子里住着另一个人  他比我先眨眼
[Pre]   钟塔倒数  走廊尽头  门锁从里面转动
[Chorus] 谁的影子  缝在我脚底  走不掉  甩不开
[Bridge] 其实我只是站在窗前看雨  但雨看起来像在看我
```

## 人格 3：周杰伦自填词系 `jay-self`

### 核心写作 DNA

**句法结构**：口语直觉 × 生活流
- 接近说话节奏：像跟朋友讲一件小事，不修饰
- 短句为主：5-8 字居多，不写长复句
- 语气词自然出现：可以用"吧"、"呢"、"嘛"（方文山系和黄俊郎系禁用）
- 半句话留白：说一半不说了，让旋律接

**意象选取规则**：
- 偏好日常生活物件：球鞋、便当、脚踏车、MP3、手写信、校门口
- 甜系意象：棒棒糖、气泡水、向日葵、游乐园、秋千
- 校园 / 青春场景：走廊、操场、图书馆角落、放学路上
- 不用古典意象，不用暗黑意象

**叙事手法**：
- 第一人称独白：100% 是"我"在讲
- 线性叙事：时间正序，按经历推进
- 对话感：偶尔直接对"你"说话（"你知道吗"、"我跟你说"）
- 副歌是心声直说：不用意象包装，直接表达

**押韵策略**：
- 松韵优先：不追求精密，自然就好
- 偏好开口韵（a/ang）：明亮、青春、释放
- 口语韵：用"啊/呀/吧"做韵脚也可以
- 不刻意押韵：如果押韵导致句子变扭，宁可不押

**用词密度**：
- 意象密度：低（每两句 1 个物件就够）
- 形容词自由：可以用"很"、"好"、"超"
- 零文言词：完全现代口语
- 重复允许度高：同一个词在 V1 和 V2 重复出现是风格，不是缺陷

### 段落分工调整

| 段落 | 周杰伦自填词系偏好 | 与默认 Lyric Builder 的差异 |
|---|---|---|
| Verse | 讲一件小事，像日记第一段 | 默认要求意象锚定，自填词系允许纯叙事 |
| Pre | "然后呢"式的推进，口语化过渡 | 默认用画面推进，自填词系用语气推进 |
| Chorus | 直说心情，可以用"我喜欢你"这类直白句 | 默认要求含蓄，自填词系允许直白 |
| Bridge | 一个小小的转折想法，像突然想到的一句话 | 默认是大转折，自填词系是小感悟 |

### 示例句风格参考（原创，非引用）

```
[Verse] 放学后你走左边那条路  我假装顺路
[Pre]   其实我家在反方向  但你不知道吧
[Chorus] 我喜欢你笑的时候  眼睛会变成月亮的形状
[Bridge] 后来我终于走了右边那条路  发现其实也不远
```

## 自动匹配决策表 `auto` 模式

当用户未指定词人人格时，根据以下维度自动匹配：

| 判断维度 | → fang-wenshan | → huang-junlang | → jay-self |
|---|---|---|---|
| 主题含东方/历史/古典 | **强匹配** | - | - |
| 主题含暗黑/悬疑/哲学 | - | **强匹配** | - |
| 主题含校园/恋爱/日常 | - | - | **强匹配** |
| 意象系统 = C（东方审美） | **强匹配** | - | - |
| 意象系统 = B（城市夜景）+ 小调 | 弱匹配 | **中匹配** | - |
| 意象系统 = A（青春回忆） | - | - | **强匹配** |
| 用户说"画面感"/"电影感" | **中匹配** | **中匹配** | - |
| 用户说"直白"/"简单"/"轻松" | - | - | **强匹配** |
| style_weight < 50 | 降权所有匹配 | 降权所有匹配 | 降权所有匹配 |

**冲突解决**：当多个人格匹配度相近时，按 `fang-wenshan > jay-self > huang-junlang` 的默认优先级（因为方文山系覆盖面最广）。若用户给了明确参数矛盾，向用户确认。

## 人格混合模式

用户可以指定两种人格的混合比例：

```
lyricist_persona: { primary: "fang-wenshan", secondary: "jay-self", ratio: "70:30" }
```

**混合规则**：
- 句法结构：跟 primary
- 意象选取：primary 为主，secondary 补充 1-2 个意象
- 押韵策略：跟 primary
- 用词密度：取两者中间值
- 段落分工：Verse/Chorus 跟 primary，Bridge 可以切到 secondary 风格（制造反差）

---

## 与其他模块的接口

- **输入来源**：`input_schema.json` 的 `lyricist_persona` 字段
- **输出影响**：调整 `lyric_builder.md` 的所有输出维度（意象库、押韵、句法、示例句）
- **编排器调用**：`original_song_generator.md` 在步骤 4 调 lyric_builder 时同时加载本模块
- **相似度自检**：`similarity_guard.md` 需要根据词人人格调整检测阈值（方文山系的意象密度上限比默认高）
